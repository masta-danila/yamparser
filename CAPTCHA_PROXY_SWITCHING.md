# Система переключения прокси при обнаружении капчи

## Описание

Реализована автоматическая система переключения прокси при обнаружении капчи на Яндекс.Картах. При появлении капчи программа автоматически:

1. **Закрывает текущий браузер** с проблемным прокси
2. **Удаляет старый профиль** браузера (если он временный)
3. **Создает новый временный профиль** браузера
4. **Выбирает новый прокси** из списка доступных
5. **Открывает новый браузер** с новым прокси и профилем
6. **Загружает ту же страницу** для продолжения парсинга

## Основные функции

### `handle_captcha_with_proxy_switching()`
**Файл:** `driver_manager.py`

Главная функция для обработки капчи с переключением прокси:

```python
def handle_captcha_with_proxy_switching(driver, proxy_manager, device_type="desktop", 
                                       profile_path=None, max_proxy_attempts=3):
    """
    Обработка капчи с автоматическим переключением прокси и созданием нового профиля
    
    Args:
        driver: текущий драйвер с капчей
        proxy_manager: менеджер прокси
        device_type: тип устройства
        profile_path: путь к профилю (будет игнорирован, создается новый)
        max_proxy_attempts: максимальное количество попыток с разными прокси
    
    Returns:
        Новый драйвер с другим прокси и новым профилем или None
    """
```

### `handle_captcha_automatically()`
**Файл:** `page_handler.py`

Обновленная функция автоматического решения капчи с поддержкой переключения прокси:

```python
def handle_captcha_automatically(driver, proxy_manager=None, device_type="desktop", 
                                profile_path=None):
    """
    Автоматическое решение CAPTCHA с возможностью переключения прокси
    
    1. Сначала пытается решить капчу автоматически
    2. Если не получается - переключает прокси и профиль
    
    Returns:
        (новый_драйвер, успех) - кортеж с драйвером и статусом
    """
```

## Преимущества новой системы

1. **Полная смена идентификации**: Новый прокси + новый профиль = новая "личность"
2. **Автоматическое восстановление**: Парсинг продолжается без участия пользователя
3. **Множественные попытки**: До 3 попыток с разными прокси
4. **Очистка ресурсов**: Старые профили автоматически удаляются
5. **Совместимость**: Работает со всеми существующими парсерами

## Логика работы

```
Обнаружена капча
        ↓
Попытка автоматического решения
        ↓
Не удалось? → Переключение прокси
        ↓
1. Сохранить текущий URL
2. Закрыть старый драйвер
3. Удалить старый профиль
4. Создать новый профиль
5. Выбрать новый прокси
6. Создать новый драйвер
7. Загрузить сохраненный URL
8. Проверить наличие капчи
        ↓
Капча исчезла? → Продолжить парсинг
        ↓
Капча осталась? → Повторить с другим прокси
```

## Настройки

В функции `handle_captcha_with_proxy_switching()` можно настроить:

- `max_proxy_attempts` - максимальное количество попыток (по умолчанию 3)
- `device_type` - тип устройства ("mobile" или "desktop")

## Безопасность

- Система **никогда не работает без прокси** - это критически важно для безопасности
- Каждый новый драйвер создается с **новым временным профилем**
- Старые профили **автоматически удаляются** для экономии места
- Поддерживается **до 40 прокси** одновременно
