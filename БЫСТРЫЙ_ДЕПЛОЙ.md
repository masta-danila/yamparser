# 🚀 Быстрый деплой YamParser на сервер

## В двух словах

1. **Скопируйте папку** на сервер
2. **Запустите один скрипт** для установки всего
3. **Замените credentials.json** на реальный
4. **Запустите парсер**

---

## 📦 1. Перенос на сервер

### Вариант A: через scp (рекомендуется)
```bash
# Из локальной папки проекта
tar -czf yamparser.tar.gz --exclude=venv --exclude=data --exclude=logs .
scp yamparser.tar.gz user@your-server:/home/user/

# На сервере
ssh user@your-server
cd /home/user/
tar -xzf yamparser.tar.gz
mkdir yamparser && tar -xzf yamparser.tar.gz -C yamparser
cd yamparser
```

### Вариант B: через git
```bash
# На сервере
git clone <your-repo-url> yamparser
cd yamparser
```

---

## ⚡ 2. Автоматическая установка

```bash
# Делаем скрипт исполняемым и запускаем
chmod +x deploy_server.sh
./deploy_server.sh install
```

**Что произойдет:**
- ✅ Установятся все системные зависимости (Python 3.12, PostgreSQL, Chromium, etc.)
- ✅ Создастся виртуальное окружение
- ✅ Установятся Python пакеты из requirements.txt
- ✅ Настроится PostgreSQL база данных
- ✅ Создастся systemd сервис для автозапуска
- ✅ Создастся скрипт управления `run_parser.sh`

---

## 🔑 3. Настройка credentials.json

```bash
# Замените заглушку на реальный файл Google Sheets API
nano credentials.json

# Или скопируйте с локальной машины
scp credentials.json user@your-server:/path/to/yamparser/
```

---

## 🎮 4. Запуск

```bash
# Запустить как сервис (работает постоянно)
./run_parser.sh start

# Или запустить один раз для теста
./run_parser.sh run-once
```

---

## 📊 5. Мониторинг

```bash
# Статус сервиса
./run_parser.sh status

# Логи в реальном времени
./run_parser.sh logs

# Логи приложения
./run_parser.sh logs-app
```

---

## 🚨 Если что-то не работает

### Проверить что установилось
```bash
# Python версия
python3 --version

# PostgreSQL
sudo systemctl status postgresql

# ChromeDriver
chromedriver --version

# Chromium
chromium --version || chromium-browser --version
```

### Запустить диагностику
```bash
# Полная диагностика
echo "=== Статус ==="
sudo systemctl status yamparser
echo "=== Логи ==="
tail -20 logs/yamparser.log
echo "=== Ошибки ==="
tail -20 logs/yamparser_error.log
```

### Тестировать компоненты
```bash
# Тест без сервиса
./run_parser.sh run-once

# Тест парсера отдельного URL
./run_parser.sh run-reviews "https://yandex.ru/maps/org/test/123/reviews/"

# Войти в виртуальное окружение
./run_parser.sh shell
```

---

## ⚙️ Все команды управления

```bash
./run_parser.sh start       # Запустить сервис
./run_parser.sh stop        # Остановить
./run_parser.sh restart     # Перезапустить
./run_parser.sh status      # Статус
./run_parser.sh logs        # Системные логи
./run_parser.sh logs-app    # Логи приложения
./run_parser.sh run-once    # Разовый запуск
./run_parser.sh shell       # Войти в venv
```

## 📋 Требования к серверу

**Минимум:**
- Ubuntu 20.04+ / CentOS 7+ / Debian 10+
- 2 ГБ RAM (рекомендуется 4 ГБ)
- 10 ГБ свободного места
- sudo права

**Поддерживаемые ОС:**
- ✅ Ubuntu 20.04, 22.04, 24.04
- ✅ CentOS 7, 8, 9
- ✅ Debian 10, 11, 12
- ✅ Amazon Linux 2
- ✅ RHEL 8, 9

## 🎯 Результат

После успешной установки:
- Парсер будет автоматически запускаться при старте сервера
- Будет обрабатывать все URL из ваших Google Sheets таблиц
- Сохранять результаты в PostgreSQL базу данных
- Логировать все операции в папку `logs/`

**Время установки:** ~5-10 минут в зависимости от скорости интернета
